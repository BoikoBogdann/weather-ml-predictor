# 🌧️ Прототип сервісу інтелектуального прогнозування опадів за допомогою ML

Вебсервіс на базі штучного інтелекту, який автоматизує збір метеорологічних даних у будь-якій точці планети, проводить порівняльний аналіз трьох алгоритмів машинного навчання, підбирає оптимальні гіперпараметри та візуалізує ймовірність появи опадів на майбутній період.

## 🏛️ Архітектура та структура файлів проєкту

Проєкт побудовано за модульним принципом чистий код (Clean Code), де інтерфейс, дата-пайплайн та скрапінг повністю ізольовані один від одного:

* **`app.py`** — Головне ядро інтерфейсу користувача (UI/UX). Відповідає за мапу Folium, інтерактивні вкладки (Таби), відображення графіків Plotly, валідацію дат та генерацію кнопок експорту в CSV.
* **`pipeline.py`** — Аналітичне серце проєкту. Тут відбувається Feature Engineering (генерація ознак), хронологічний розподіл даних на блоки Train/Val/Test (60/20/20), динамічне зважування незбалансованих класів, крос-валідація гіперпараметрів та фінальне навчання моделей.
* **`data_scrapper.py`** — Автономний клієнт для роботи з API Open-Meteo. Містить інтелектуальну логіку автоматичного перемикання базових ендпоінтів (`/v1/archive` або `/v1/forecast`) залежно від часового вікна, заданого користувачем.
* **`requirements.txt`** — Специфікація залежностей проєкту, необхідна для автоматичного розгортання та ізоляції контейнера в хмарі (Production environment).

## 🧠 Фізичне обґрунтування та відбір ознак (Feature Selection)

Для усунення ефекту **мультиколінеарності** (лінійної залежності ознак) та уникнення **витоку даних (Data Leakage)**, з датасету було повністю вилучено сирі параметри опадів (`precipitation_sum`, `rain_sum`). 

Замість них сформовано розширений **«золотий набір» із 10 незалежних синоптичних та часових ознак**, які описують повний термодинамічний та баричний стан атмосфери:

1.  **`shortwave_radiation_sum` (МДж/м²):** Показник сумарної сонячної радіації. Різке падіння вдень фіксує суцільну фронтальну хмарність.
2.  **`sunshine_duration` (секунди):** Чистий час сонячного сяйва. Прямий нелінійний маркер затягнутого неба.
3.  **`daylight_duration` (секунди):** Астрономічна тривалість світлового дня (задає сезонний контекст).
4.  **`apparent_temperature_max` (°C):** Показник максимальної відчувальної температури (індекс паркості та енергетичного потенціалу конвективних хмар).
5.  **`temperature_2m_mean` (°C):** Середня добова температура повітря на висоті 2 метрів (визначає вологомісткість маси).
6.  **`wind_speed_10m_max` (км/год):** Максимальна швидкість вітру для відстеження баричних градієнтів.
7.  **`wind_gusts_10m_max` (км/год):** Різкі шквали та пориви вітру, притаманні динамічним грозовим фронтам.
8.  **`wind_direction_10m_dominant` (градуси):** Домінантний вектор вітру, що вказує на походження повітряної маси (морська чи континентальна).
9.  **`et0_fao_evapotranspiration` (мм):** Рівень потенційного випаровування вологи за стандартом ФАО, що формує об'єм пари в тропосфері.
10. **`dayofyear` (1–365):** Числовий день року для нелінійного моделювання сезонних хвиль.

## ⚔️ Аналітичний батл алгоритмів (Моделі)

Сервіс проводить Hold-out тестування на трьох різних архітектурних класах:

* **Logistic Regression** — Лінійна базова модель. Використовує математичну функцію сигмоїди для перетворення зваженої суми ознак у ймовірність:
    $$P(y=1) = \frac{1}{1 + e^{-z}}$$
* **Random Forest** — Ансамбль незалежних дерев рішень. Рахує ймовірність як пряму частку "голосів" дерев (наприклад, якщо 75 дерев зі 100 проголосували за дощ, ймовірність = 74%). Найбільш стійка модель до шумів та дисбалансу класів.
* **HistGradientBoosting** — Послідовний ансамбль слабких дерев, де кожне наступне дерево мінімізує помилки (градієнт функції втрат) попереднього за допомогою оптимізованого гістограмного підходу.

## 🛡️ Аналітичні інсайти тестування в різних кліматичних зонах

Проєкт підтримує глобальне тестування, що дозволяє виявити фундаментальні закономірності поведінки метрик:
* **Помірно-континентальний клімат (Україна):** Моделі демонструють збалансовані метрики ($F1\text{-score} \approx 0.70-0.80$), оскільки класи опадів та сухої погоди відносно рівномірні протягом року. Лідером за важливістю є `sunshine_duration`.
* **Аридний / Пустельний клімат (Саудівська Аравія):** Спостерігається парадокс **Extreme Class Imbalance** (екстремального дисбалансу класів). Метрика **ROC AUC злітає до $0.96-0.97$**, що доводить колосальну роздільну здатність відібраних ознак. Водночас **F1-Score просідає до $0.40-0.50$**, оскільки рідкісність події різко штрафує математичну точність ($Precision$) за будь-яку хибну тривогу через пустельні бурі.
